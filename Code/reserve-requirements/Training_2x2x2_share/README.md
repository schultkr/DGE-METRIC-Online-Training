# ToyModelSOEMC Training Package (Self-Contained)

This folder is a self-contained MATLAB + Dynare package for sharing the ToyModelSOEMC training setup.
It includes all scripts needed to run the model from this folder without relying on sibling project folders.

## Quick Start

1. Open MATLAB.
2. Change directory to this folder.
3. Run:

```matlab
ToyModelSOEMC_run_training_2x2x2
```

## Automated Export Run (Console + Figures)

To run the model and automatically export artifacts for reporting, run:

```matlab
ToyModelSOEMC_run_training_with_exports
```

This creates a timestamped folder under:

- `Presentation/run_outputs/YYYYMMDD_HHMMSS/`

with:

- `matlab_console.log` (full diary/console output)
- `export_manifest.txt` (index of exported figures)
- figure exports in PNG, PDF, and FIG formats
- `workspace_snapshot.mat` (workspace state at end of run)

## What This Training Version Changes

The training launcher sets:

- `MINIMAL_MODE = false`
- `TRAINING_MODE = true`

When `TRAINING_MODE=true`, Step 4 (joint policy search) in `ToyModelSOEMC_joint_optimal.m` uses a reduced 2x2x2 grid:

- `d_grid = [90, 365]`
- `phi_R_grid = [0, 1.0]`
- `phi_IQ_grid = [0.04, 0.08]`

This produces 8 policy grid points instead of the full 72 points.

## Folder Contents: Every File and Purpose

### 1) Entry scripts

- `ToyModelSOEMC_run_training_2x2x2.m`

  - Main entry point for sharing/training.
  - Sets training flags and launches the master runner.
  - Use this script when teaching or demoing with reduced runtime.
- `ToyModelSOEMC_run_training_with_exports.m`

  - Automation wrapper for reproducible reporting.
  - Runs the training launcher, records a MATLAB diary log, and exports all open figures.
  - Writes artifacts to a timestamped folder in `Presentation/run_outputs/`.
- `ToyModelSOEMC_run.m`

  - Master workflow script.
  - Executes the full sequence:
    - Dynare compile and initialization
    - Steady-state diagnostics
    - Deterministic crisis demo
    - Growth and Energy Master Plan scenarios
    - Step 4 joint optimization (`run('ToyModelSOEMC_joint_optimal.m')`)
  - Contains `MINIMAL_MODE` behavior and waitbar progress updates.

### 2) Core model specification

- `ToyModelSOEMC.mod`

  - Dynare model file (equations, variables, shocks, simulation blocks).
  - Source of the model structure solved by Dynare.
  - Do not rename unless you update calls in the runner.
- `ToyModelSOEMC_steadystate.m`

  - Custom steady-state routine used by Dynare and policy scripts.
  - Computes steady-state values and writes calibrated capacity bars (`Q_c_bar`) back to parameters.
  - Saves `ToyModelSOEMC_steadystate.mat` when run.

### 3) Joint optimization and stochastic components

- `ToyModelSOEMC_joint_optimal.m`

  - Step 4 policy search over reserve stock (`d`), drawdown intensity (`phi_R`), and capacity investment (`phi_IQ`).
  - Uses `TRAINING_MODE` / `MINIMAL_MODE` to pick grid size and MC count.
  - Runs Monte Carlo draws under PF mode and writes results summary to `joint_optimal_results.txt`.
- `ToyModelSOEMC_shock_gen.m`

  - Generates exogenous shock paths for each MC draw (PF mode).
  - Includes price innovation shocks (`nu_c`) and supply disruption shocks (`e_c_shock`).
  - Supports optional override struct for custom shock parameters.
- `ToyModelSOEMC_pf_homotopy_solve.m`

  - Robust PF solve helper using continuation/homotopy.
  - Solves sequentially at scaled shock levels (default 5 steps) before full shock amplitude.
  - Used to improve convergence in difficult paths.

### 4) Local helper utilities (included to avoid external dependencies)

- `get_ss.m`

  - Returns a variable steady-state value by name from `oo_.steady_state`.
- `extract_var.m`

  - Extracts a simulated endogenous variable path by name from Dynare outputs.
- `compute_welfare.m`

  - Computes welfare metric:
    - `sum_{t=0}^{T-1} beta^t * log(C_t)`
  - Includes guard against non-positive consumption values.

### 5) Documentation

- `README.md`
  - This file.

## Runtime Artifacts Generated by the Scripts

When you run the training launcher, the scripts may generate/update files in this folder:

- `ToyModelSOEMC_steadystate.mat`

  - Saved by `ToyModelSOEMC_steadystate.m`.
- `joint_optimal_results.txt`

  - Saved by `ToyModelSOEMC_joint_optimal.m` with optimal policy summary.
- Dynare-generated artifacts

  - Depending on Dynare version/settings, additional output files can appear (logs, temporary/generated files).

## Output Files Reference (Clear Explanations)

This section explains every output file type you should expect after running the package.

### A) Main model output files (package root)

- `joint_optimal_results.txt`
  - Created by `ToyModelSOEMC_joint_optimal.m`.
  - Human-readable summary of the Step 4 optimum (e.g., `d_star`, `phi_R_star`, `phi_IQ_star`, welfare values, cost metrics, MC metadata).
  - Use this file first when you want final policy numbers without opening MATLAB.

- `ToyModelSOEMC_steadystate.mat`
  - Created by `ToyModelSOEMC_steadystate.m`.
  - Stores steady-state vector (`ys`) and parameter vector (`params`) used by the model.
  - Useful for debugging calibration and reproducing a solved steady state.

- `ToyModelSOEMC.log`
  - Created by Dynare.
  - Text log of Dynare preprocessing/solve messages and diagnostics.
  - Use this when model compilation or solver steps fail.

### B) Export-run output files (under `Presentation/run_outputs/<timestamp>/`)

- `matlab_console.log`
  - Created by `ToyModelSOEMC_run_training_with_exports.m` via MATLAB `diary`.
  - Captures console prints for the run. May be empty if diary did not capture session output.

- `export_manifest.txt`
  - Index of all exported figures and their file paths.
  - Includes run timestamp and success flag.

- `NN_<FigureName>.png`
  - Raster image export of each MATLAB figure (good for slides/docs).

- `NN_<FigureName>.pdf`
  - Vector export of each MATLAB figure (best for print and LaTeX scaling).

- `NN_<FigureName>.fig`
  - Native MATLAB figure file (editable in MATLAB after export).

- `workspace_snapshot.mat`
  - Workspace dump at end of export run.
  - Useful for post-run analysis without rerunning the model.

### C) Presentation document output files (under `Presentation/`)

- `ToyModelSOEMC_outputs_explained.tex`
  - Source LaTeX documentation explaining model outputs.
  - Also embeds explicit figure evidence from a chosen run folder.

- `ToyModelSOEMC_outputs_explained.pdf`
  - Compiled report from the `.tex` file.
  - Share this when you want the narrative + visuals in one document.

- `ToyModelSOEMC_outputs_explained.aux`, `.log`, `.out`, `.fls`, `.fdb_latexmk`, `.synctex.gz`
  - LaTeX build byproducts.
  - These are intermediate files used by LaTeX tools; they can be regenerated.
  - Keep them for iterative editing, but they are optional for distribution.

### D) Optional/generated folders you may see

- `ToyModelSOEMC/` and `+ToyModelSOEMC/`
  - Dynare-generated code folders/files for the model.
  - Contents depend on Dynare version and run settings.
  - Usually treated as generated artifacts.

## Execution Flow (High Level)

1. `ToyModelSOEMC_run_training_2x2x2.m`
2. `ToyModelSOEMC_run.m`
3. `dynare ToyModelSOEMC noclearall` (reads `ToyModelSOEMC.mod` and steady-state routine)
4. Scenario steps in `ToyModelSOEMC_run.m`
5. `ToyModelSOEMC_joint_optimal.m` for 2x2x2 training search

## What You Can Safely Customize

- Training grid values in `ToyModelSOEMC_joint_optimal.m` under `if TRAINING_MODE`.
- Number of homotopy steps in `ToyModelSOEMC_pf_homotopy_solve.m` callers (`PF_HOMOTOPY_STEPS`).
- Shock override parameters passed to `ToyModelSOEMC_shock_gen.m`.

## Requirements

- MATLAB
- Dynare (the scripts try common Windows paths such as `C:\dynare\7.0\matlab`)
